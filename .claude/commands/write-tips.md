# Tips記事を書く

TerminalHub の機能を「使い方記事」として Notion に書き、Discord #tips で紹介するまでのフロー。
「〇〇の記事書いて」「Tips書いて」等で発動。引数でお題（機能名）が指定されればそれを、なければ `docs/tips-draft.md` から未記事化のネタを提案して選んでもらう。

## 前提

- 対象読者は「そんなことができると知らない人」。困りごと起点ではなく **結論・ベネフィット先行** で書く
- 実装や手順は記事化の前に必ずコードや実環境で裏取りする（アイコン名・コマンド・ポート番号などを推測で書かない）

## 手順

### 1. 記事作成（Notion MCP）

Notion の「Claude 作業スペース > TerminalHub 機能紹介」（page_id `39369d5bd86b81c997c1ed57eb79c651`）配下に子ページとして作成する。

記事の型:

1. **こんな事ができるよ！** — 冒頭でベネフィットを見せる。タイトルも結論先行（例:「TerminalHub をスマホでそのまま使う — Tailscale で5分」）
2. **やり方詳細** — 方針の概要（箇条書き数行）だけ書き、具体的な作業は「以下を Claude Code などに貼り付けてください」という **AI伴走用の依頼文（コードブロック）** にする
   - 依頼文には注意点・禁止事項（例: funnel は使わない）も織り込み、読み飛ばされても事故らないようにする
   - ユーザー自身の操作（スマホ側など）が必要な場合も、AI が案内する項目として依頼文に含め「繋がるところまで一緒に」型にする
3. **注意点など** — 要点のみ箇条書き。最後は「うまくいかなければ状況を AI に伝えて一緒に調整してください」で締める
4. 末尾に対象バージョンを1行

スクリーンショットは後から人手で入れるため、入れ所に「`> 📷 **スクショ挿入**: （何の画面か）`」形式のプレースホルダーを置く。

作成できたら、**作成レスポンスに含まれる編集用URL（`https://app.notion.com/p/...` のフル）をそのままユーザーに提示する**。ページIDだけの短縮形や「ページID: xxxx」表記にはしない（ユーザーが自分で探す羽目になる）。ユーザーがクリック一発でスクショ挿入などの編集に入れるようにするため。手順2で導出する公開URL（`gilded-pruner-343.notion.site/...`）とは別物なので、**両方ともフルURLで併記する**。

### 2. 公開URLの導出と確認

公開URLは以下で組み立てる（ページIDは手順1の作成レスポンスから控えておく。スラッグ省略可）:

```
https://gilded-pruner-343.notion.site/<ページID(ハイフン無し)>
```

まずこのURLが記事に解決されるか確認する。ハブページごと Web 公開済みなら子ページは自動的に公開されているので、解決すればそのまま手順3へ。

解決しない場合のみ、ユーザーに「共有 > Webで公開」をオンにしてもらう（公開設定は MCP から変更できない）。それでも導出URLが解決しなければ、ドメインが変わった可能性があるのでユーザーに公開URLを貼ってもらい、新ドメインをこのファイルに反映する。

### 3. Discord投稿

投稿文（記事の要約2〜3行＋絵文字1つ＋URL）を提示してユーザーの承認を得てから、#tips チャンネル（ID `1522925874591039518`）へ `mcp__discord__discord_execute` の `messages.send` で投稿する。

### 4. ネタ帳の更新

`docs/tips-draft.md` の該当ネタの行末に、次の書式で記事化済みの印を付ける（未記事化ネタの判定はこの印の有無で行う）:

```
→ ✅記事化済み: <公開URL>
```
