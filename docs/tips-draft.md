# TerminalHub Tips 候補リスト（ドラフト）

Discord などでの「Tips 投稿」用ネタ候補。週1〜隔週ペースでの発信を想定。
精査して採用/不採用、追記してください。

---

## 🟦 セッション基本

1. **同じセッションを別ブラウザ/タブで開ける** — Circuit 毎管理なので、PC とサブモニタで同じターミナルを表示できる
2. **セッションは「開いたとき」に起動する** — 遅延初期化なので、10 個並べてもバックグラウンドは静か
3. **ターミナル内の URL はクリックできる** — WebLinksAddon。`http://localhost:5173` みたいなのは即ブラウザへ
4. **入力履歴は永続化される** — 上下キーで過去入力。アプリ再起動後も残る
5. **タイトルバーに処理状態が出る** — 経過時間・トークン数がリアルタイム

## 🟩 Claude Code / Gemini / Codex 連携

6. **CLI の出力を解析して「処理中/完了」を自動判定** — Claude Code・Gemini・Codex 各対応
7. **5 秒間出力がないとハング判定** — タイマーで自動検出
8. **処理完了でベル通知** — 非アクティブセッションだけ鳴る（作業中のは黙る）
9. **トークン使用量が見える** — Claude/Gemini の使用量をセッション一覧でひと目
10. **Codex CLI も普通に使える** — Claude/Gemini と同じ枠で

## 🟧 Git Worktree

11. **セッションから Worktree を作れる** — 親ディレクトリの **兄弟** として作成（中ではなく横）
12. **Worktree ごとに別 Claude Code を走らせる運用** — 同じリポジトリの別ブランチを並列で触れる

## 🟨 タスクランナー / npm

13. **package.json を自動検出して npm scripts がタスクになる** — `npm run dev` をボタン化
14. **タスクは独立セッション** — 普通のターミナルと別管理。ログを残したまま別作業へ
15. **タスク選択状態は LocalStorage に保存** — 再起動後も同じタスク画面に戻る

## 🟥 通知 / Webhook / Hook

16. **Webhook 通知（処理開始・完了）を Slack/Discord に飛ばせる** — eventType=start/complete、JSON ペイロード
17. **閾値秒数を超えた処理だけ通知** — 短い処理で通知爆撃にならない
18. **Hook API (`/api/hook/claude/{sessionId}`)** — Claude Code 側 hook と直結。SessionStart/Stop/Notification を受信
19. **Webhook ペイロードに folderPath が入る** — どの作業ディレクトリの完了かが分かる

## 🟪 MQTT 遠隔起動（隠れ目玉）

20. **VPS 経由で外出先から PC のターミナルを起動できる** — MQTT 経由のリモートトリガ
21. **暗号化は RSA-OAEP + AES-256-GCM** — nonce リプレイ防止つき。本気の作り
22. **パスワード設定で他人からの起動はブロック** — SHA256 ハッシュ照合

## 🟫 起動モード / インストール

23. **App Mode（Chrome `--app`）でデスクトップアプリ風に起動** — `TerminalHub-App.bat`。タブバーなしで集中できる
24. **通常ブラウザモードもある** — `TerminalHub.bat`。既定ブラウザで開く
25. **インストーラー版は .NET ランタイム不要** — 自己完結 publish

## 🟦 トラブル時/小ワザ

26. **ログは `%LOCALAPPDATA%\TerminalHub\logs`** — エラー追跡はここ
27. **エラーレポート Webhook** — 例外発生時に自分宛 Discord に飛ばせる
28. **フォルダ選択ダイアログは最前面に出る** — v1.0.58 で修正済（App Mode で裏に隠れる問題）
29. **長い文字列を貼っても切れない** — ConPTY 書き込みを 265 文字チャンクで明示フラッシュ
30. **複数 TerminalHub プロセスを起動可能** — 開発インスタンスと通常利用を併存できる
